1.数据结构

2.String与[]byte转换

  • 标准转换
  • 强转换

string与[]byte在底层结构上是非常的相近（后者的底层表达仅多了一个cap属性，因此它们在内存布局上是可对齐的）

强转换比标准转化性能更好，原因在于强转换直接将String底层的数组指针转为Slice指针，省去了内存拷贝的过程。

3.String 拼接

  • 使用 +

  • 使用 strings.Builder

  • 使用[]byte

综合易用性和性能，一般推荐使用 strings.Builder 来拼接字符串。与预分配内存的 []byte 相比，因为省去了 []byte 和字符串(string) 之间的转换，内存分配次数还减少了 1 次，内存消耗减半。

比较 strings.Builder 和 +: 当使用 + 拼接 2 个字符串时，生成一个新的字符串，那么就需要开辟一段新的空间，新空间的大小是原来两个字符串的大小之和。而 strings.Builder，bytes.Buffer，包括切片 []byte 的内存是以倍数申请的。
