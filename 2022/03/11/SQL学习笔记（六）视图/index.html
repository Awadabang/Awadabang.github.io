SQL学习笔记（六）视图

发表于2022-03-11|更新于2022-03-29
|阅读量:

1.视图介绍

SQL VIEW 语法

View（视图）是虚拟的表，在数据库中不实际存在，视图只保存了查询的SQL逻辑，不保存结果

2.视图检查选项

使用WITH CHECK OPTION创建视图，MySQL会在对视图进行插入、更新、删除时检查是否符合视图创建时的定义。

  • WITH CASCADED CHECK OPTION （默认） 级联，会无条件检查是否满足上层视图条件
  • WITH LOCALCHECK OPTION 会根据上一层视图的检查选项进行检查

3.视图的更新

要使视图可更新，视图中的行于基础表中的行之间存在一对一关系。如果视图包含以下的任何一种情况，则视图不可更新

  • 聚合函数或窗口函数（SUM(), MIN(), MAX(), COUNT()等）
  • DISTINCT
  • GROUP BY
  • HAVING
  • UNION或者UNION ALL

4.视图的作用

  • 简单 视图可以简化用户对数据的理解，也可以简化操作
  • 安全 数据库可以授权，但不能授权到数据库特定行和特定列上，通过视图用户只能查询和修改他们能见到的数据
  • 数据独立 可以帮助用户屏蔽真实表结构变化带来的影响。比如真实表列名变化了，视图中可以仍然指定为原名，用户无感
